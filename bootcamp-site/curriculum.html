System Architecture
Syllabus

A 24-week engineering residency designed to break you down and rebuild you as a systems thinker. No "Hello World" tutorials here.

Weeks 1-4

Computer Science Fundamentals

We start by peeling away the abstractions. You will write C++ to understand memory management, pointers, and data structures. You will implement hash tables and linked lists from scratch.

Modules

  • • Memory Management & Pointers (C++)
  • • Big O Notation & Complexity Analysis
  • • Data Structures (Trees, Graphs, Hash Maps)
  • • Sorting & Searching Algorithms

Project

Custom Shell: Build a POSIX-compliant shell in C. Handle processes, signals, and pipes.

Weeks 5-12

Distributed Backend Systems

Move to high-level architecture. Design scalable REST and GraphQL APIs. Understand database normalization, indexing, and caching strategies using Redis.

Modules

  • • Node.js Event Loop & Streams
  • • PostgreSQL (ACID, Normalization, Indexing)
  • • Docker & Containerization
  • • Authentication (JWT, OAuth2, Sessions)

Project

E-commerce API: A high-performance API handling 10k requests/sec. Implements rate limiting, caching, and background job processing.

Weeks 13-20

Frontend Engineering

Not just "making it pretty". We focus on client-side architecture, state management patterns, performance optimization, and accessibility.

Modules

  • • React Internals (Reconciliation, Virtual DOM)
  • • State Management (Redux Toolkit, Context)
  • • TypeScript Generics & Utility Types
  • • Web Performance (Lighthouse, Lazy Loading)

Project

Real-time Dashboard: Analytics dashboard with live websocket data, complex data visualization, and optimistic UI updates.

Weeks 21-24

Capstone & DevOps

The final sprint. You pitch, design, and build a full-scale application. Then you deploy it using industry-standard DevOps pipelines.

Modules

  • • AWS (EC2, S3, RDS, Lambda)
  • • CI/CD Pipelines (GitHub Actions)
  • • System Design Architecture Interviews
  • • Infrastructure as Code (Terraform)

Capstone

Production Launch: Your final project must be live, accessible via custom domain, and handle real user traffic.
